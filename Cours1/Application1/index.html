Application 1

La préparation et l'import de tables sous SAS

Remarques préliminaires

Un fichier Excel doit être préparé en amont de l'import

Il est extrêmement rare de trouver des bases de données qui soient formatées pour SAS à peine extraites

La présentation du fichier

La première ligne du fichier peut contenir les noms de variables

Les noms de variables sous SAS ont des spécificités qu'il faudra TOUJOURS respecter

Les noms de variables

Les noms de variables comprennent 32 caractères maximum

 

Les noms sont composés de lettres non accentuées, éventuellement de chiffres et du symbole "_"

 

Tout autre caractère spécial est à proscrire (@, espace | $ £ ! ...)

 

Les noms de variables ne doivent jamais commencer par un chiffre

 

Les noms de variables doivent être uniques par table

Les noms de variables

Par défaut, l'assistant utilise la première ligne pour définir les noms de colonnes

Pour signaler l'absence de noms de variables dans la première ligne à SAS, deux actions selon le mode d'import doivent être opérées

Dans le cas de l'assistant

Au moment de l'étape de sélection de la feuille à importer décrite dans le cours d'introduction, il faut aller dans les options

Dans le cas de l'assistant

Puis décocher l'option par défaut de "Noms des colonnes en première ligne"

Dans le cas de la proc import

Dans la proc import décrite dans le cours d'introduction il conviendra d'attribuer la valeur NO à l'option GETNAMES

proc import

...

  GETNAMES = NO ;

...

RUN;

Les noms de variables

Dans ce cas les variables seront nommées (F1, ... , Fn), avec n le nombre de colonnes non vides dans le fichier importé

La présentation du fichier

Chaque ligne doit correspondre à un individu

 

Il convient d'être vigilant sur la présence de sous totaux ou de total dans un tableau, SAS l'interprêtera comme un individu à part entière

 

Il est fréquent d'être confronté à des observations manquantes dans une base de données

 

SAS gère les valeurs manquantes différemment selon que la variable soit numérique ou alphanumérique

Gestion des valeurs manquantes

Une variable numérique manquante sera identifiée par un .

Une variable alphanumérique manquante sera identifiée par "", ou " "

ATTENTION

Une variable numérique manquante sera toujours la plus petite valeur d'une variable numérique

Ainsi une condition "inférieur ou égal" sera toujours respectée par une valeur manquante et ne sera pas toujours désirable

La présentation du fichier

Les modalités ne doivent pas respecter les même règles que les noms de variables

 

Cependant elles présentent leurs propres règles qu'il conviendra de respecter sous peine de problème lors de l'import

 

Les plupart du temps, les problèmes d'import ne sont pas signalés par le système et sont découverts après plusieurs traitements

 

La plus grande prudence est à observer car l'import est une étape cruciale à ne pas sous estimer

Problématique des formats

La principale caractéristique sur laquelle il faudra être prudent est le format et plus particulièrement la longueur des variables

Bien qu'une variable numérique soit par défaut définie comme une variable de taille 8 (la plus grande possible sur SAS), la longueur des variables alphanumériques est par défaut définie comme la longueur de la plus longue chaine de caractère d'une variable lors de l'import

Cela peut mener à des problèmes de troncatures décrits dans l'introduction en cas de concaténation de tables ou de modification de modalités

Remarque

Si une transposition est nécessaire, les modalités deviennent des variables et se doivent de respecter les règles décrites plus haut

Il convient dès lors d'éviter les modalités qui contiennent des espaces ou autres caractères spéciaux dans cette éventualité

Maitrise de l'import de données

Partie 1

Soit le fichier excel produit-interieur-brut-reg-1990-2011.xls

Nouvelle version : PIB_1990_2015_regions_diffusion.xls

Ouvrir le fichier (onglet PIB en valeur 1990-2015) et

expliquer en quoi le format du fichier ne convient pas à un import sous SAS

Réponses Partie 1

  • Les noms de colonnes se trouvent à la ligne 4
  • Une ligne vide est présente à la ligne 5
  • Présence de cellules non vides en ligne 1, 29 et 30
  • Les noms de variables sont des chiffres
  • Année et Région contiennent des caractères accentués
  • Présence de sous-totaux et total général dans le tableau


On décide de corriger dans Excel directement

A l'issue de la correction le fichier produit-interieur-brut-reg-1990-2015 formaté.xls peut être importé

Partie 2

Définir la librairie Prelim qui servira à stocker la table à importer

 

Libname Prelim 'C:\...\SAS FILES';

PROC IMPORT

  OUT = Prelim.PIB_REG

  DATAFILE = 'C:\...\produit-interieur-brut-reg-1990-2015 formaté.xls'

  DBMS = XLS REPLACE;

SHEET = 'PIB en valeur 1990-2011$';

GETNAMES = YES;

RUN;

Remarques Partie 2

Le journal renvoie la note suivante si le programme a été correctement rédigé

PRELIM.PIB_REG was successfully created.

La table est alors stockée dans la librairie Prelim et est accessible depuis l'onglet Explorateur

Il convient de toujours vérifier que l'import s'est bien passé en ouvrant la base de données après l'import
